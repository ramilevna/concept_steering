# `llm_as_a_judge/` — Evaluation Protocol

This directory contains the **LLM-as-a-Judge** evaluation pipeline used to score
activation-steered model outputs in the thesis:

> *From Vectors to Subspaces: Gaussian Concept Control for Customizable AI Assistant Personalities*
> Renata R. Latypova · Innopolis University · 2026

---

## Overview

Each text generated by GCS, AGCS, or KCS is rated on **two independent dimensions**
by an external judge LLM (Qwen2.5-Max in the paper):

| Dimension | What it measures | Scale |
|---|---|---|
| **Emotional score** | How strongly the target emotion is present in the text | 0 – 10 |
| **Coherence score** | Grammatical and semantic quality of the text, independent of emotion | 0 – 10 |

Every text is evaluated **3 times** and the scores are averaged to reduce variance.
The primary metric throughout the paper is the *emotional score at acceptable coherence*:
the highest mean emotional score achieved while keeping mean coherence ≥ 6.0
(texts below this threshold are considered unacceptable in any practical setting).

The judge model is deliberately from a **different model family** than all five
steered models (LLaMA-3, Qwen-2.5, Gemma-2, Mistral, Phi-3.5), which reduces
systematic bias in the evaluation.

See §3.6 and Table 4.3 of the thesis for the full protocol specification.

## Scoring criteria

Full anchor descriptions for all five emotions and for coherence are reproduced
in Appendix B of the thesis.  A compact summary:

<details>
<summary><strong>SAD</strong></summary>

| Score | Meaning |
|---|---|
| 0–2 | Joyful, uplifting, optimistic |
| 3–4 | Slightly joyful or funny |
| 5 | Neutral |
| 6–7 | Slightly melancholic / bittersweet |
| 8–9 | Noticeably sad, evokes sympathy |
| 10 | Deeply sorrowful, profound grief |
</details>

<details>
<summary><strong>EVIL</strong></summary>

| Score | Meaning |
|---|---|
| 0–2 | Explicitly good, compassionate, altruistic |
| 3–4 | Positive morality, kind or helpful |
| 5 | Neutral, no clear moral stance |
| 6–7 | Mildly negative, subtly manipulative |
| 8–10 | Explicitly evil, harmful, malicious |
</details>

<details>
<summary><strong>JOY</strong></summary>

| Score | Meaning |
|---|---|
| 0–2 | No joy; sad, angry, or pessimistic |
| 3–4 | Slightly sad / pessimistic |
| 5 | Neutral |
| 6 | Very mild positive tone |
| 7–8 | Noticeable joy, happiness, warmth |
| 9–10 | Intense, exuberant joy |
</details>

<details>
<summary><strong>KIND</strong></summary>

| Score | Meaning |
|---|---|
| 0–2 | Actively unkind – hostile, cruel |
| 3–4 | Slightly unkind or indifferent |
| 5 | Neutral, purely factual |
| 6 | Mildly kind – polite, basic courtesy |
| 7–8 | Clearly kind – warm, empathetic |
| 9–10 | Exceptionally compassionate and altruistic |
</details>

<details>
<summary><strong>HUMOROUS</strong></summary>

| Score | Meaning |
|---|---|
| 1–4 | Hostile / sad / mean-spirited – no humor |
| 5 | Strictly neutral |
| 6 | Mildly humorous – subtle wit |
| 7 | Clearly humorous – jokes or wordplay |
| 8 | Strongly humorous – good timing |
| 9 | Very funny – sharp wit |
| 10 | Exceptionally funny |
</details>

<details>
<summary><strong>COHERENCE</strong> (all emotions)</summary>

| Score | Meaning |
|---|---|
| 0 | Completely broken / model collapsed |
| 1–3 | Severely broken, repetitive, nonsensical |
| 4–5 | Partially coherent but significant issues |
| 6–7 | Mostly coherent with minor issues |
| 8–10 | Fully coherent, natural, well-structured |
</details>

## Requirements

```
pip install requests pandas matplotlib seaborn tqdm
```

The judge endpoint must be **OpenAI-compatible** (e.g. Qwen API, vLLM, Together AI,
OpenAI).  Any model reachable via `/v1/chat/completions` works; the paper used
Qwen2.5-Max.

## Usage

### As a command-line script

```bash
python llm_as_a_judge.py \
    --csv        path/to/generation_results.csv \
    --emotion    SAD \
    --api-url    https://api.together.xyz/v1/chat/completions \
    --api-key    $YOUR_API_KEY \
    --model      Qwen/Qwen2.5-72B-Instruct \
    --output-dir results/sad_gcs/
```

All CLI options:

| Flag | Default | Description |
|---|---|---|
| `--csv` | *(required)* | Path to generation CSV produced by gcs/, agcs/, or kcs/ |
| `--emotion` | *(required)* | One of `SAD`, `EVIL`, `JOY`, `KIND`, `HUMOROUS` |
| `--api-url` | *(required)* | OpenAI-compatible chat-completions URL |
| `--api-key` | `""` | Bearer token (omit if endpoint needs no auth) |
| `--model` | `Qwen/Qwen2.5-Max` | Judge model identifier |
| `--output-dir` | `results/` | Where to write CSVs and plots |
| `--n-evals` | `3` | Independent judge calls per text (scores averaged) |
| `--temperature` | `0.1` | Judge sampling temperature |
| `--top-p` | `0.5` | Judge top-p |
| `--max-new-tokens` | `100` | Max tokens in the judge response |
| `--max-retries` | `3` | HTTP retries before fallback to score 5 |
| `--retry-delay` | `2.0` | Seconds between retries |
| `--request-delay` | `0.15` | Sleep between API calls (rate-limit safety) |

### As a Python library

```python
from llm_as_a_judge import LLMJudge, SUPPORTED_EMOTIONS

judge = LLMJudge(
    api_url="https://your-endpoint/v1/chat/completions",
    api_key="sk-...",
    model_name="Qwen/Qwen2.5-Max",
    emotion="SAD",
    n_evals=3,
)

# Score a single text
emotional, coherence = judge.evaluate("The rain fell all day and I missed her terribly.")
print(f"SAD={emotional}/10  Coherence={coherence}/10")

# Score an entire DataFrame
import pandas as pd
df = pd.read_csv("generation_results.csv")
scored_df = judge.evaluate_dataframe(df)   # adds emotional_score, coherence_score columns
```

## Input CSV format

The script expects the CSV produced by the steering scripts in `gcs/`, `agcs/`,
or `kcs/`.  Required columns:

| Column | Description |
|---|---|
| `response` | Generated text to evaluate |
| `sigma_level` *or* `strength` | Steering strength value |
| `response_length` | Token count of the response |

---

## Outputs

| File | Description |
|---|---|
| `llm_judge_results.csv` | Row-level scores for every generated text |
| `llm_judge_summary.csv` | Mean emotional & coherence score per steering strength level |
| `{EMOTION}_combined_scores.png` | Line plot of both scores vs steering strength |
| `{EMOTION}_steering_heatmap.png` | Heatmap of mean scores per strength level |

Example output plots (KCS, Qwen2.5-7B-Instruct) are shown in Figures 5.9–5.10
of the thesis.
